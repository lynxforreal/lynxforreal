## Research Focus Areas

### Offensive Security Research

I actively research how modern offensive techniques operate in real environments, with emphasis on **behavioral execution rather than surface‑level exploits**.

My work includes:

* Studying real malware families and offensive frameworks from a defensive‑analysis perspective
* Understanding execution flow, staging, and runtime behavior
* Mapping attacker actions to detection opportunities

---

### Malware Analysis & Development (Research‑Driven)

* Analysis of malware written in C/C++ to understand structure and design decisions
* Building minimal research implementations to validate concepts
* Comparing different implementation strategies and their behavioral footprint

This work is aimed at **learning attacker methodology**, not reproducing full malware campaigns.

---

### Reverse Engineering

* Static analysis to understand program logic and control flow
* Dynamic analysis to observe runtime behavior
* Identifying how binaries interact with the operating system

Tools used include industry‑standard reverse engineering and debugging platforms.

---

### Windows Internals & Execution Model

* Researching how user‑mode programs interact with the operating system
* Understanding process and thread lifecycles
* Studying executable formats and loader behavior

The goal is to reason about *why* certain techniques work and *how* they become visible to defensive tooling.

---

### EDR & Detection Awareness

> **EDR (Endpoint Detection and Response):** Security solutions designed to monitor endpoint behavior, detect malicious activity, and enable response actions.

My focus is on:

* Understanding behavioral detection logic
* Learning how offensive techniques are identified in practice
* Evaluating trade‑offs between stealth, reliability, and complexity

This knowledge is applied to research and analysis, not misuse.
